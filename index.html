TOPIC 12

Why Different Programming Languages?

💡 Just like no single tool builds an entire house, no single programming language is best for every problem. This topic explores why different languages exist and which kinds of problems they are best suited for.

The Core Idea

Programming languages are designed with specific goals in mind — speed, readability, portability, safety, or ease of learning. The appropriateness of a language to an application is the key question every developer must ask before starting a project.

Application Areas & Their Best-Fit Languages

  • Web Applications — JavaScript is the backbone of the front-end; Python (Django/Flask), PHP, Ruby, and Java are common back-end choices. HTML/CSS structure and style content.
  • Games — C++ dominates AAA game development (Unreal Engine) for raw speed. C# powers Unity. Performance and memory control are critical.
  • Formula Translation / Scientific Computing — Fortran (the original "Formula Translation") and Python (NumPy, SciPy) excel at heavy mathematics. MATLAB is also used in academia.
  • Mobile Applications — Swift and Objective-C for iOS; Kotlin and Java for Android. Cross-platform tools like Flutter (Dart) or React Native (JavaScript) let one codebase run on both.

Why Can't One Language Do Everything?

  • Performance vs. Ease of Use — C/C++ run very fast but are harder to write safely. Python is easy but slower.
  • Platform Requirements — Mobile OSes have preferred or required languages.
  • Domain-Specific Features — SQL exists purely for database queries. HTML is for document structure.
  • Community & Libraries — A language with great libraries for a task (e.g., Python for ML) is often the practical choice even if another language could do it.

Comparing Concepts Across Languages

🎯 Focus on variable declarations and control structures across C, C++, Java, JavaScript, and Python. The concepts are the same — the syntax is what changes.

Variable Declarations

A variable declaration tells the computer to reserve memory and give it a name. Languages differ in how strictly you must specify the data type.

Control Structure: If / Else

Click a language tab to see the syntax.

// C – if / else
int score = 75;
if (score >= 50) {
    printf("Pass\n");
} else {
    printf("Fail\n");
}
// C++ – if / else
int score = 75;
if (score >= 50) {
    cout << "Pass" << endl;
} else {
    cout << "Fail" << endl;
}
// Java – if / else
int score = 75;
if (score >= 50) {
    System.out.println("Pass");
} else {
    System.out.println("Fail");
}
// JavaScript – if / else
let score = 75;
if (score >= 50) {
    console.log("Pass");
} else {
    console.log("Fail");
}
# Python – if / else  (no curly braces – indentation matters!)
score = 75
if score >= 50:
    print("Pass")
else:
    print("Fail")

Control Structure: For Loop

Counting from 1 to 5.

// C
for (int i = 1; i <= 5; i++) {
    printf("%d\n", i);
}
// C++
for (int i = 1; i <= 5; i++) {
    cout << i << endl;
}
// Java
for (int i = 1; i <= 5; i++) {
    System.out.println(i);
}
// JavaScript
for (let i = 1; i <= 5; i++) {
    console.log(i);
}
# Python  (range end is exclusive, so range(1,6) gives 1–5)
for i in range(1, 6):
    print(i)

Control Structure: While Loop

Keep looping while a condition is true.

// C
int count = 1;
while (count <= 3) {
    printf("count = %d\n", count);
    count++;
}
// C++
int count = 1;
while (count <= 3) {
    cout << "count = " << count << endl;
    count++;
}
// Java
int count = 1;
while (count <= 3) {
    System.out.println("count = " + count);
    count++;
}
// JavaScript
let count = 1;
while (count <= 3) {
    console.log("count = " + count);
    count++;
}
# Python
count = 1
while count <= 3:
    print(f"count = {count}")
    count += 1

Key Observations

  • Python uses indentation instead of curly braces { } — get the spacing wrong and the program breaks.
  • C, C++, Java, JavaScript use curly braces to group blocks of code.
  • C and C++ require strict data types — you must say int, float, char when declaring a variable.
  • JavaScript uses let / const / var; the type is inferred automatically.
  • Python needs no keyword at all — just write x = 5 and it figures out the type.
  • Java is the most verbose — every program needs a class, a main method, and explicit types.

LAB 1

Language Scavenger Hunt (No coding needed)

🎯 Objective: Match real-world applications to programming languages.
  1. Write down 5 apps or websites you use (e.g., Instagram, a game, Google Maps).
  2. Research what language each one was primarily built with. A quick search like "what language is Instagram built in" works fine.
  3. For each, write one reason why that language makes sense for that application.
  4. Compare your list with a classmate — did you find any surprises?
💬 Discussion: Were any languages used for more than one type of application on your list? What does that tell you?
LAB 2

Variable Declaration Explorer

🎯 Objective: Declare and print variables in at least TWO of the five languages.
  1. Go to OnlineGDB. Use the language dropdown (top right) to switch between Python and JavaScript.
  2. In Python, declare: your name as a string, your age as an integer, and your GPA as a float. Print all three.
  3. Repeat the same in JavaScript using let. Use console.log() to print.
  4. Try the same in Java if you have time — you'll need to wrap it in a class and main method.
  5. Write down: What's the biggest difference you noticed between Python and the others?
💬 Tip: Python doesn't need a data type keyword — just name = "Alex". JavaScript needs let name = "Alex";. Java needs String name = "Alex";.
LAB 3

Grade Checker — 3 Ways

🎯 Objective: Write the same if/else logic in Python, JavaScript, and Java.
  1. The logic: A student's mark is 68. If it's ≥ 70, print "Distinction". If it's ≥ 50, print "Pass". Otherwise, print "Fail".
  2. Write this in Python using if / elif / else.
  3. Write it in JavaScript — same logic, but with curly braces and semicolons.
  4. Optional challenge: Write it in Java — you'll need the full class wrapper.
  5. Run each version and confirm they all print "Pass".
💬 Python tip: Python uses elif (not else if). JS and Java use else if.
LAB 4

Times Table Generator

🎯 Objective: Use a for loop to generate a times table in Python and one other language.
  1. Write a Python program that asks the user for a number, then prints its times table from 1 to 12. Use input() to get the number and a for loop to print each line. Example output: 5 x 3 = 15
  2. Now write the same in JavaScript using a for loop. You can hardcode the number since JS in the browser console doesn't have easy input().
  3. Identify: which line of code is the most different between the two versions?
💬 Python f-string hint: print(f"{n} x {i} = {n*i}")
LAB 5 · CHALLENGE

Language Comparison Report

🎯 Objective: Synthesise everything from Topics 12 & 13 into a short written or presented comparison.
  1. Choose two languages from the five studied.
